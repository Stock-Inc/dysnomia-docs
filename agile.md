## Цель и принципы
Цель процесса — предсказуемо поставлять инкременты продукта короткими итерациями (спринтами), сохраняя прозрачность статуса задач и качества.  
Принципы: маленькие поставки, единый бэклог, приоритет от продукта.

## Роли и ответственность
- Product Manager: формирует приоритеты, принимает результат спринта, отвечает за ценность.  
- Team Lead / DevOps: архитектурные решения, технические стандарты, инфраструктура, релизный контур.  
- Project Manager / Scrum‑функция: помогает с планированием, следит за ритуалами, снимает блокеры, ведет метрики.  
- Разработчики (frontend/backend/mobile/bot): оценивают, берут задачи в работу, несут ответственность за реализацию и качество.

## Спринты и ритуалы
- Длительность спринта: 1–2 недели (выберите фиксированно и не меняйте без причины).  
- Sprint Planning (в начале): уточнение цели спринта, перенос задач из бэклога в спринт, оценка и разбор рисков.  
- Daily (10–15 минут): синхронизация по прогрессу/блокерам, без глубоких обсуждений (их выносить в отдельные созвоны).  
- Sprint Review (конец): демонстрация инкремента и сбор фидбэка от продуктовой стороны.  
- Retrospective (после review): 1–3 конкретных улучшения процесса/техпрактик в следующий спринт.

## Работа в Jira (Scrum board)
**Артефакты**
- Product Backlog: единый список задач, упорядоченный по приоритету.
- Sprint Backlog: выбранные задачи + план достижения цели спринта.

**Типы задач (рекомендуемо)**
- Epic (крупная цель/фича), Story (пользовательская ценность), Task (техзадача), Bug

**Статусы (пример)**
- Backlog → Selected for Sprint → In Progress → Code Review → Done.

## Интеграция Jira ↔ GitHub (правила разработки)
Jira можно интегрировать с GitHub так, чтобы ветки/коммиты/PR автоматически связывались с задачами Jira.  [support.atlassian](https://support.atlassian.com/jira-cloud-administration/docs/integrate-jira-software-with-github/)  
Чтобы связь работала стабильно, включайте ключ задачи Jira (например, `DYS-123`) в **название ветки**, сообщение коммита или заголовок/описание PR — тогда эта активность появится в блоке Development у задачи в Jira.  [github](https://github.com/atlassian/github-for-jira)

**Соглашения (рекомендуемо)**
- Branch naming: `DYS-123-short-title`, `DYS-456-fix-login`, `DYS-789-deps`.
- Commit message: `DYS-123: implement message sync`.
- PR:
  - В заголовке — ключ Jira.
  - В описании — кратко: что сделано.

**Flow работы**
1) Задача в Jira переводится в In Progress и назначается исполнителю.  
2) Создается ветка по соглашению, ведется разработка, коммиты содержат ключ.  
3) Открывается PR → Code Review → исправления → merge в main.  
4) Задача переводится в Done только после выполнения (включая обновление документации/миграций при необходимости).
